1. Introduction

COnVIDa is a tool developed by the Cybersecurity and Data Science Laboratory at the University of Murcia (Spain) that allows easily gathering data related to the COVID19 pandemic form different data sources, in the context of Spain, and visualize them in a graph. Contact us at convida@listas.um.es.

2. How to use COnVIDa

In order to use this tool, first we have to select the dates range for which we want to collect data.

Next we choose the Spanish region or regions we are interested in.

Finally, we just select those data items within each data source (COVID19, INE, Mobility, MoMo and AEMET) that we want to know and all those data will be automatically plotted in the main graphs, as well as the corresponding summary tables of such data. It is worth mentioning that when passing the mouse over each data item, a description of such data item will be shown.

COnVIDa offers two visualization modes: temporal and by regions. In the temporal visualization, the graph will show the daily values for those temporal data items available. For instance, if we select the COVID19 cases in Murcia from 21/02/2020 to 18/05/2020, the X axis will show all the days between those two dates, while Y axis will show the daily COVID19 cases in Murcia. In turn, in the regional visualization graph the data will be aggregated by those regions that we have previously selected. Some data items (for instance those of INE) can only be plotted in the regional visualization graph. However, in order to plot temporal data items in the regional visualization graph COnVIDa will use the mean of those data items within the specified data ranges and will show them for each region chosen by the user.

Once the data items are plotted, it is possible to easily change the scale of the graph, either linear, or logarithmic. The logarithmic scale is useful to simultaneously visualize data series with different orders of magnitude.

Additionally, it is also possible to change the graph type itself, choosing between a lines graph and a bars graph.

Last but not least, the summary table shows, as its own name states, a statistical summary of each of the selected data items, including: a data items count, their mean, their standard deviation, their minimum, their percentile 25, their median, their percentile 75 and their maximum value.

As we can see, two buttons are offered to download, either all the data items collected according to the criteria specified by the user, or the entire summary table. COnVIDa offers the possibility to download any of thes two tables in the formats of CSV, XLS, JSON and HTML.

3. Data sources

Current version of COnVIDa includes 5 data sources related to the COVID19 pandemic in Spain. These data sources are:

  1. COVID19: The first source of data from which to obtain relevant information about the COVID19 pandemic in Spain are all those data items actually associated to sch pandemic published by Datadista in Github (who in turn retrieves such data from the Ministerio de Sanidad and the Instituto de Salud Carlos III). From here COnVIDa takes the data items of recovered, cases, PCR confirmed, test confirmed, deaths, hospitalized and ICU.
  2. INE: The second relevant data source considered in COnVIDa is INE (Instituto Nacional de Estadística). From this source some remarkable data items are collected such as physical activity, body mass index (BMI), tobacco consumption, household by family type, households by occupation density and over 65 years old alone.
  3. Mobility: The next data source included in COnVIDa has to do with the citizens' mobility. In this case the original data sources are actually two: Google COVID-19 Community Mobility Reports and Apple COVID‑19 - Mobility Trends Reports. From the first data source we obtain the citizens' mobility in different spaces such as: i) grocery and pharmacy, ii) parks, iii) residential, iv) retail and recreation, v) transit stations and vi) workplace. In turn, from the second data source we retrieve the citizens' mobility while driving their vehicles.
  4. MoMo: Another data source with remarkable information about the pandemic is the mortality monitoring system MoMo handled by the Instituto de Salud Carlos III. From this source we obtain the daily observed deaths for each region, as well as the lower and upper bounds of such series. Additionally, the daily expected deaths per region are also collected, as well as the 1st and 99th percentiles of such series.
  5. AEMET: Finally, COnVIDa also includes meteorological data stemming from the AEMET (Agencia Estatal de Meteorología). In particular, this tool allows querying daily values about rainfall, maximum pressure, minimum pressure, maximum gust, isolation, maximum temperature, mean temperature, minimum temperature, wind speed, altitude and gust direction.

As stated previously, when passing the mouse over each data item, a description of such item will be automatically displayed.

4. Source code

COnVIDa has been developed from its very conception as an Open Science project with the aim and spirit of serving and assisting anyone who might need it in the context of the COVID19 pandemic in Spain. In this regard, all the project source code is publicly accessible through the next repository, where a developer manual is also included:

https://github.com/CyberDataLab/COnVIDa

4. Future versions

In future versions of COnVIDa we expect:

  • Allowing the possibility of sharing a URL with the results obtained from a particular query.
  • Adding more tables from INE with economical, social and demographic data.
  • Implementing a REST API to conduct automatic queries to the COnVIDa server.
  • Implementing epidemiological models to predict the evolution of the pandemic.
  • Allowing users authentication to keep last queries, etc.
  • Allowing users to upload their own CSV to compare it with the data collected by COnVIDa.
  • Adding Google Trends as a new data source.
